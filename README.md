# RAG-система 
**Автор:** Маташков Фома, 25.М81-мм 

## Описание
Реализует Retrieval-Augmented Generation (RAG) систему, которая:
• ищет релевантные документы в векторном хранилище (FAISS);

• фильтрует их по компаниям, упомянутым в вопросе;

• формирует контекст и отправляет запрос в LLM (DeepSeek);

• возвращает ответ в формате JSON с указанием источника (чанка);

• автоматически формирует `submission.json` и отправляет его на сервер.

## Описание основных компонентов 
### `config.py` 
Содержит: 
• пути к файлам и директориям; 

• путь к FAISS-индексу; 

• API-ключ для доступа к GigaChat. 

Важно!  API-ключ необходимо указать вручную. 

--- 
### `main.py` 
Основной исполняемый скрипт, который: 
1. загружает вопросы; 
2. выполняет поиск релевантных чанков в FAISS; 
3. формирует контекст для LLM; 
4. отправляет запрос в модель GigaChat; 
5. валидирует и приводит ответы к требуемому типу; 
6. формирует `submission.json`; 
7. отправляет файл на сервер. 

## Инструкция по запуску

### 1. Клонирование репозитория

```bash
git clone https://github.com/alternativeEnding/RAG_MF
cd RAG
```

### 2. Ключ API
В файле config.py необходимо указать API-ключ GigaChat:
```API_KEY='<КЛЮЧ>'```

### 3. Подготовка окружения и установка зависимостей
Создаём виртуальное окружение и устанавливаем зависимости из requirements.txt
```bash
python -m venv venv
venv\Scripts\activate   

pip install -r requirements.txt
```
### 4. Для корректного запуска main.py должны быть подготовлены следующие данные:

Файл с вопросами ```data/questions.json```
Векторное хранилище FAISS ```data/faiss_index/```

### 6. Последовательность запуска (полный пайплайн)
- vector_store.py        # построение FAISS-индекс
- main.py                # запуск RAG-системы и формирование submission
